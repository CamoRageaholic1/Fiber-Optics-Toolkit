# Contributing to Fiber Optics Toolkit

Thank you for your interest in contributing!

## Guidelines

### Code Contributions

1. **Fork the repository**
2. **Create a feature branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. **Make your changes**
   - Follow existing code style
   - Add docstrings to functions
   - Include type hints

4. **Test your changes**

5. **Update documentation**
   - Update README.md if adding new tools
   - Add examples for new features

6. **Commit with clear messages**
   ```bash
   git commit -m "Add: New DWDM wavelength calculator"
   ```

7. **Submit pull request**

### Standards Compliance

All contributions must comply with:
- **FOA (Fiber Optic Association)** standards
- **TIA/EIA** telecommunications standards
- **IEC** international standards

### Code Style

- Follow PEP 8
- Use descriptive variable names
- Include units in names (e.g., `loss_db`, `length_km`)

### Documentation

- All functions need docstrings
- Include parameter types and return types
- Add usage examples

## Bug Reports

Include:
- Tool/module affected
- Input parameters
- Expected vs actual output
- Python version and OS

## Feature Requests

- Describe the use case
- Explain value
- Reference relevant standards

## Priority Areas

- Additional OTDR formats
- International standards
- More examples
- Performance optimizations

## Review Process

1. All PRs reviewed by maintainer
2. CI must pass
3. Documentation updated
4. Standards verified
